5. Expected i2cget Output (after daemon running)
--------------------------------------------------------------------------------

$ sudo i2cget -y 0 0x2A 0x00 w
0x1e00  ( HID descriptor length = 30, LE )

$ sudo i2cget -y 0 0x2A 0x02 w
0x0200  ( Input register address )

$ sudo i2cset -y 0 0x2A 0x04 0x01  # Write RESET command to command register
$ sudo i2cget -y 0 0x2A 0x05 w      # Read from data register (response)

8. Error Scenarios
--------------------------------------------------------------------------------

ERROR: i2c-dev not loaded
-----------------------------
[Error] Failed to open I2C bus. Please check:
  1. i2c-dev kernel module loaded: modprobe i2c-dev
  2. i2c-stub loaded with address: modprobe i2c-stub chip_addr=0x2A
  3. Permission to access /dev/i2c-0
  4. Try running as root or: sudo chmod 666 /dev/i2c-0

ERROR: i2c-stub not loaded
-----------------------------
[Error] Failed to set slave address 0x2A: Device or resource busy

ERROR: Address in use
-----------------------------
[Error] Failed to set slave address 0x2A: Device or resource busy
-> Another device is using this address. Unload it first:
   sudo modprobe -r i2c-stub

ERROR: Permission denied
-----------------------------
[Error] Failed to open /dev/i2c-0: Permission denied
-> Run as root: sudo ./vhidi2c-daemon
-> Or add user to i2c group: sudo usermod -aG i2c $USER
