# 第1章：Windows vs Linux — 從 Windows 工程師的角度理解 Linux

> **本章目標**：理解 Linux 是什麼，了解它和 Windows 的根本差異，知道常見的 Linux 發行版

---

## 🎯 先問一個問題

你知道你每天使用的 Windows 作業系統是誰做的嗎？

答案是：**微軟（Microsoft）**。這是一家公司的產品。

那 Linux 呢？

**Linux 不是一家公司的產品**。它是全世界成千上萬的工程師共同開發的開源（Open Source）專案。任何人可以看到它的原始碼，任何人可以貢獻程式碼，任何人可以免費使用。

---

## 🏠 什麼是 Linux？（用 Windows 的角度理解）

### 想像一下

| Windows | Linux |
|---------|-------|
| 由微軟公司統一開發、維護 | 由社群共同開發，沒有單一老闆 |
| 需要付費（授權費） | 免費（大部分發行版） |
| 原始碼不公開 | 原始碼公開，所有人都能看 |
| 只在少數硬體上執行 | 几乎所有硬體都能執行 |
| 驅動程式通常自動安裝 | 有時需要手動安裝驅動 |
| 桌面市場佔有率高 | 伺服器市場佔有率極高 |

### 專業術語時間

**開源（Open Source）**：程式的「食譜」是公開的，所有人都能看、能改、能散布。

> 就像一家餐廳把招牌料理的食譜公開，誰都能做這道菜，甚至能改進它。

**發行版（Distribution）**：Linux 只是核心（Kernel），不同組織或社群把 Linux 核心 + 各種軟體包在一起，就變成不同的「發行版」。就像 Android 手機，不同品牌用同一個 Android 系統，但介面和預裝軟體不同。

## 🧩 Linux 的組成

Linux 作業系統由幾個部分組成：

```
┌─────────────────────────────────────────────┐
│           Linux 作業系統整體                 │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────┐  ← 應用程式層              │
│  │  Firefox    │     （你用的軟體）           │
│  │  VS Code    │                            │
│  └─────────────┘                            │
│                                             │
│  ┌─────────────┐  ← 命令列工具層            │
│  │   bash      │     （終端機、Shell）        │
│  │   ls, cd    │                            │
│  └─────────────┘                            │
│                                             │
│  ┌─────────────┐  ← GNU 工具層              │
│  │  gcc        │     （編譯器、核心工具）       │
│  │  make       │                            │
│  └─────────────┘                            │
│                                             │
│  ┌─────────────┐  ← Linux 核心（Kernel）     │
│  │             │     （真正的「大腦」）        │
│  │  記憶體管理  │                            │
│  │  程序管理   │                            │
│  │  硬體驅動   │                            │
│  │             │                            │
│  └─────────────┘                            │
│                                             │
└─────────────────────────────────────────────┘
```

> **為什麼要了解這個？**
> 當你使用 `gcc` 編譯程式時，你用到的是 GNU 工具鏈，不是 Linux 本身。
> 所以正確的說法是「Linux + GNU」，有些人稱之為「GNU/Linux」。
> 但日常對話中，大家都直接說「Linux」。

## 🌟 常見的 Linux 發行版

### Ubuntu（新手最推薦）

**類比 Windows 的話**：Ubuntu 像 Windows 10/11 — 容易上手、友善的圖形介面、軟體豐富

- **開發公司**：Canonical（一家公司，但 Linux 本身是開源的）
- **特色**：最容易上手、文件完整、社群龐大
- **適合**：第一次接觸 Linux 的新手、桌面使用、伺服器
- **官網**：https://ubuntu.com/

### Debian

**類比 Windows 的話**：Debian 像 Windows 的「專業版」— 穩定、但介面較樸實

- **特色**：最穩定、軟體包最多、完全由社群維護
- **適合**：需要高度穩定性的伺服器、進階使用者
- **衍生版本**：Ubuntu 就是從 Debian 來的
- **官網**：https://www.debian.org/

### Fedora

**類比 Windows 的話**：Fedora 像 Windows 的「 Insider Preview 」— 走在前面、測試新技術

- **開發公司**：Red Hat（已被 IBM 收購）
- **特色**：新技術、新功能更新最快
- **適合**：開發者、想嘗試新技術的人
- **衍生版本**：Red Hat Enterprise Linux (RHEL)
- **官網**：https://fedoraproject.org/

### 其他值得知道的發行版

| 發行版 | 特色 | 適合對象 |
|--------|------|----------|
| **Linux Mint** | Ubuntu 為基底，介面更像 Windows | 從 Windows 轉換的人 |
| **CentOS** | RHEL 的免費版（已停止，改為 Stream） | 企業伺服器 |
| **Arch Linux** | 從零開始手動安裝，高度客製化 | 進階使用者 |
| **Raspbian/Raspberry Pi OS** | 為樹莓派設計 | 嵌入式、IoT |

> **Fedora 和 Red Hat 的關係**
> - Fedora 是 Red Hat 贊助的「實驗場」，新技術先在 Fedora 測試
> - 技術成熟後，會進入 Red Hat Enterprise Linux (RHEL)
> - CentOS 原本是 RHEL 的免費克隆版，但現在變成 CentOS Stream（滾動更新版）

## 🖥️ 為什麼伺服器都用 Linux？

### 1. 免費 💰

Windows Server 需要購買授權，一台伺服器可能貴達數千元。而 Linux 免費。

> 就像餐廳老闆可以免費拿到五星級食譜，當然選免費的。

### 2. 穩定 ⚡

Linux 伺服器可以連續運作幾年不關機、不重開機。Windows 伺服器需要偶爾重啟。

### 3. 安全 🔒

 Linux 的權限機制更嚴格。病毒很難感染 Linux（因為它設計時就考慮了安全性）。

### 4. 可客製化 🔧

你想把 Linux 改成什麼樣都可以。Windows 是「我給你什麼就用什麼」，Linux 是「你想要什麼就裝什麼」。

### 5. 效能 🚀

 Linux 可以跑在非常低階的硬體上。一台 10 年前的電腦裝 Linux 還能跑，但裝 Windows 10 可能會很辛苦。

### 6. 雲端支援 ☁️

所有主要雲端平台（AWS、Google Cloud、Azure）都有 Linux 伺服器選項。

## 📊 Windows 和 Linux 核心差異

### 使用者視角

| 比較項目 | Windows | Linux |
|----------|---------|-------|
| 設計哲學 | 圖形介面優先 | 命令列優先（但現在也有好用的圖形介面） |
| 軟體安裝 | 安裝程式 (.exe/.msi) 或 Microsoft Store | 套件管理器（一行指令就安裝） |
| 系統更新 | Windows Update（有時強制重開機） | 可以邊跑邊更新，不用重開機 |
| 檔案路徑 | `C:\Program Files\` | `/usr/bin/` |
| 硬碟分割 | C:、D: 磁碟機 | 統一成一個目錄樹 |
| 預設管理員 | Administrator | root |
| 軟體格式 | .exe, .msi | .deb, .rpm, 或直接原始碼 |

### 設計架構差異

**Windows**：封閉但完整，所有東西綁在一起

**Linux**：模組化，每個零件可以分開替換

```
Windows 思維：
┌──────────────────────────────────────┐
│           Windows 系統                │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │  UI    │ │ 系統   │ │ 工具   │   │
│  │(外觀)  │ │(核心)  │ │(附屬)  │   │ 全部綁在一起
│  └────────┘ └────────┘ └────────┘   │
└──────────────────────────────────────┘

Linux 思維：
┌──────────────────────────────────────┐
│           Linux 系統                  │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │  UI    │ │ 系統   │ │ 工具   │   │
│  │ GNOME  │ │ Kernel │ │ Coreutils│  │
│  │ 或 KDE │ │        │ │        │   │ 可以自由替換！
│  └────────┘ └────────┘ └────────┘   │
└──────────────────────────────────────┘
```

## 🔑 重點整理

1. **Linux 不是作業系統品牌**，它是一個「核心」，不同組織把它包裝成不同產品（發行版）

2. **開源不等於免費**，但 Linux 大部分發行版既開源又免費

3. **Ubuntu、Debian、Fedora** 是最常見的發行版，選一個開始學

4. **伺服器偏好 Linux** 是因為：免費、穩定、安全、可客製化

5. **Linux 和 Windows 的設計哲學不同**：Windows 重視圖形介面，Linux 重視命令列和模組化
